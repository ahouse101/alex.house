Lab Spend

https://www.labspend.com/

Lab Spend maintains a comprehensive suite of tools targeting research labs. I've worked with them as a full stack developer since January 2016. The current site is build on a Laravel-based PHP stack with a responsive front-end using the Materialize CSS framework and Backbone.js.

In the summer of 2016, I led the port of the core website from its original plain PHP implementation with various server-side and client-side libraries to a new, streamlined architecture. We chose to adapt the Laravel framework to the project's needs, serving partially constructed Blade views as well as exposing a REST API that a reactive Backbone-based front-end could consume. This resulted in a hybrid Laravel website with each page existing as a compartmentalized single-page JavaScript application, eliminating several server side dependencies and homogenizing our various services' codebases.

UM Supermileage

http://umsm.engin.umich.edu/

University of Michigan's Supermileage team is a small team of student engineers that designs, builds, and competes with a highly efficient vehicle, creating a new design every year. I built their current WordPress website and admin dashboard.

The original Supermileage website was based on a very old WordPress template. The team didn't have a web developer, so no real design work had gone into it. In 2015, I joined the team and started building a new website. Because it was important that other members of the team were able to upload photos and edit copy, I built upon the WordPress base with an entirely new template, as well as a streamlined Admin Dashboard. The team continues to manage the content year-by-year.

BareMiminum

https://bareminimum.codeplex.com/

BareMinimum was a project I built my senior year of high school in my spare time. It was a gradebook program that could tell you the lowest scores required on a set of assignments in order to get a given overall grade. It supported arbitrary section nesting with percentage weights, and could save a set of scenarioes as a JSON file.

I built BareMinimum with C#, using Windows Forms for the UI. I had previous Windows Forms experience, so I used an open source project called ObjectListView that wrapped the native Win32 list view. The core math functions were relatively simple, so the vast majority of the development time was spent on UI work, serialization, and compatibility with older versions of the .NET framework.

Shoebox

Shoebox is a media manager, capable of working with several video and photo formats, including some raw photo formats. It can manage media that is stored locally or in cloud storage. Features include collections, tagging, face recognition, and timestamp-based video scene markers.

Shoebox was my first major project built with Electron. It uses Vue.js for view management, FFmpeg for media manipulation, and dcraw for raw photo rendering.

Depot Data

The Tech Depot within University of Michigan's ITS department manages computer inventory and system rollout across many of the University's departments. I worked with the Depot in software rollout, eventually building the Depot Data website to track activities across the night operations performed by students. There was also an accompanying .NET app for Windows that acted as a companion app to the main website.

Due to limitations in deployment, the Depot Data website is entirely JavaScript with no server-side code. The UI was simple, built on Bootstrap and Chart.js.

FizzDevelop

https://fizzdevelop.codeplex.com/

FizzDevelop was a lightweight editor that could compile and run Java and C# programs. It supported syntax highlighting and basic project structure.

I built FizzDevelop in high school, using C# and Windows Forms. The editing component was Scintilla, integrated into Windows Forms with the open source Scintilla.NET binding, for which I actively contributed documentation.
